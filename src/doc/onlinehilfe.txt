@datei=AuswahlFrame.html
@titel=Mitglieder / Boote / Ziele / Mehrtagesfahrten / Fahrtenabzeichen / Standardmannschaften / Gruppen
@beschreibung
efa verwaltet für jedes Fahrtenbuch eine eigene Boots-, Mitglieder- und Zielliste, sowie eine Liste der Mehrtagesfahrten. Desweiteren gibt es eine Liste von Standardmannschaften für jedes Boot sowie eine Liste von Gruppen, in der beliebige Mitglieder zusammengefaßt werden können. Um diese Listen zu bearbeiten, wähle im efa-Hauptfenster aus dem Menü <i>Administration</i> den Punkt <i>Fahrtenbuch</i> und dort einen der Punkte <i>Mitgliederliste</i>, <i>Bootsliste</i>, <i>Zielliste</i> bzw. <i>Mehrtagesfahrten</i> sowie <i>Standardmannschaften</i> oder <i>Gruppen</i> aus dem Menü <i>Administration</i>.
In diesem Fenster werden alle Einträge der aktuell geöffneten Mitglieder-, Boots-, Ziel-, Mehrtagesfahrt-, Standardmannschaften- oder Gruppen-Liste angezeigt. Neue Einträge können hinzugefügt werden, vorhandene Einträge können geändert und gelöscht werden.
@erklaerung
Durch gleichzeitiges Drücken von &lt;Shift&gt; bzw. &lt;Strg&gt; und Klicken mit der linken Maustaste können mehrere Einträge gleichzeitig ausgewählt werden, die sich dann alle zusammen löschen oder bearbeiten lassen.
[but]Hinzufügen|Ein neues Mitglied / Boot / Ziel / eine neue Standardmannschaft hinzufügen (bei Mehrtagesfahrten nicht verfügbar).
[but]Bearbeiten|Die aktuell ausgewählten Einträge bearbeiten.
[but]Löschen|Die aktuell ausgewählten Einträge löschen.
[but]Liste ausgeben|Gibt eine Liste mit allen Einträgen aus.
[but]Status aktualisieren|Aktualisiert den Status aller Mitglieder anhand ihres Jahrgangs und des aktuellen Status.
[but]Schließen|Schließt das Fenster.

@datei=AuswahlFrame_Fahrtenabzeichen.html
@titel=DRV-Fahrtenabzeichen
@beschreibung
Wenn für das DRV-Fahrtenabzeichen elektronisch gemeldet werden soll, so müssen in efa die bereits erworbenen Fahrtenabzeichen aller Mitglieder erfaßt werden (egal, ob diese auf herkömmliche Art oder elektronisch erworben wurden).
Um diese Liste zu bearbeiten, wähle im efa-Hauptfenster aus dem Menü <i>Administration</i> den Punkt <i>DRV-Fahrtenabzeichen</i>.
In diesem Fenster siehst Du eine Liste mit Mitgliedern, die bereits Fahrtenabzeichen erworben haben.
Eine ausführliche Erklärung des elektronischen DRV-Fahrtenabzeichens, der elektronischen Meldung und der Verwaltung von Fahrtenabzeichen findest Du in dem Dokument <i>elektronisches DRV-Fahrtenabzeichen -- Anleitung für Vereine</i> unter <a href="http://www.nmichael.de/software/efa/efawett.html">http://www.nmichael.de/software/efa/efawett.html</a>.
@erklaerung
Durch gleichzeitiges Drücken von &lt;Shift&gt; bzw. &lt;Strg&gt; und Klicken mit der linken Maustaste können mehrere Einträge gleichzeitig ausgewählt werden, die sich dann alle zusammen löschen oder bearbeiten lassen.
[but]Hinzufügen|Für ein neues Mitglied die bereits erworbenen Fahrtenabzeichen eingeben.
[but]Bearbeiten|Fahrtenabzeichen eines bereits erfaßten Mitglieds anzeigen oder bearbeiten.
[but]Löschen|Eintrag eines Mitglieds löschen. Es wird hierbei <i>nicht</i> das Mitglied in der Mitgliederliste gelöscht, sondern lediglich seine Informationen zu den bereits erworbenen Fahrtenabzeichen.
[but]Liste ausgeben|Gibt eine Liste mit allen Einträgen aus.
[but]Bestätigungsdatei abrufen|Nachdem eine Wettbewerbsmeldung vom DRV verarbeitet ist, erstellt dieser eine Bestätigungsdatei, die die vom DRV signierten elektronischen Fahrtenhefte enthält. Diese muß vom Verein vor der nächsten Meldung abgerufen werden. Klicke hierzu bitte diesen Button.
[but]Schließen|Schließt das Fenster.
@siehe

@datei=DatensicherungFrame.html
@titel=Datensicherung
@beschreibung
efa ist in der Lage, alle relevanten, selbst erstellten Dateien in einem ZIP-Archiv zu sichern. Diese Daten können einerseits als Sicherung verwendet werden, andererseits aber auch z.B. auf einen anderen Computer übertragen werden.
Um eine Datensicherung durchzuführen, wähle im Hauptfenster von efa im Menü <i>Datei</i> den Punkt <i>Datensicherung</i>. efa ermittelt daraufhin selbständig, welche Verzeichnisse gesichert werden müssen. Im Normalfall ist diese Auswahl ausreichend, so daß keine weiteren Verzeichnisse mehr hinzugefügt werden müssen.
@erklaerung
[but]Sichern|In der Spalte <i>Sichern</i> kannst Du auswählen, welche der vorgeschlagenen Verzeichnisse gesichert werden sollen.
[but]Verzeichnis|Diese Spalte gibt das zu sichernde Verzeichnis an.
[but]Unterverzeichnisse|Wenn dieser Punkt gewählt ist, so werden für das zu sichernde Verzeichnis auch alle Unterverzeichnisse gesichert.
[but]Sicherungsdatei erstellen in|Hier kann ausgewählt werden, wo das ZIP-Archiv mit der Sicherung erstellt werden soll. Möglich ist zum einen die Erstellung im efa-Backup-Verzeichnis, zum anderen die Erstellung in einem beliebigen Verzeichnis.
[but]Datensicherung starten|Sichert alle ausgewählte Daten.
@siehe

@datei=EfaFrame.html
@titel=Hauptfenster
@beschreibung
Das Hauptfenster von efa wird beim Starten des Programms geöffnet. Von hier aus erreichst Du sämtliche Funktionen von efa. Außerdem werden in diesem Fenster die Fahrten eingegeben.
Im Menü <i>Datei</i> findest Du die Funktionen zum <a href="FahrtenbuchNeuFortsetzenFrame.html">Erstellen eines neues Fahrtenbuchs</a>, zum Öffnen, sowie zum Speichern eines Fahrtenbuchs. Außerdem gibt es einen Menüpunkt zum <a href="RestoreBackupFrame.html">Wiederherstellen von Sicherungsdateien (Backups)</a> und zum <a href="ImportFrame.html">importieren von Fahrten aus anderen Fahrtenbüchern</a>, sowie zum Beenden von efa. Außerdem kann über den Punkt <i>efa Online-Update</i> über das Internet eine neue Version von efa heruntergeladen werden.
Das Menü <i>Administration</i> bietet unter dem Punkt <i>Fahrtenbuch</i> die Möglichkeit, <a href="NeuesFahrtenbuchFrame.html">Einstellungen zum Fahrtenbuch</a> zu verändern, sowie die <a href="AuswahlFrame.html">Boots-, Mitglieder-, Ziellisten und Mehrtagesfahrten</a> zu bearbeiten. Hinter dem Punkt <i>Synonymlisten</i> verbergen sich Unterpunkte zum <a href="SynonymFrame.html">Bearbeiten von Synonymen für Boote, Mitglieder und Ziele</a>. <i><a href="AuswahlFrame.html">Standardmannschaften</a></i> erlaubt das Festlegen von Standardmannschaften für bestimmte Boote. Der Punkt <i><a href="VereinsConfigFrame.html">Elektronische Wettbewerbsmeldung</a></i> führt zu den Einstellungen für die Meldung von Teilnehmern für Wettbewerbe über das Internet. Über den Punkt <i><a href="WriteProtectFrame.html">Zugriffsschutz</a></i> können Dateien gegen unbefugte Veränderungen geschützt werden, und die <i><a href="EfaConfigFrame.html">Allgemeinen Einstellungen</a></i> bieten sonstige Konfigurationsmöglichkeiten.
Das Menü <i>Ausgabe</i> enthält als einzigen Eintrag den Menüpunkt <i><a href="StatistikFrame.html">Statistiken und Meldedateien erstellen</a></i>, über den die Erstellung von Statistiken und Meldedateien gesteuert wird.
Im Menü <i>Info</i> kann unter <i>Dokumentation</i> die Dokumentation zu efa aufgerufen werden. Der Punkt <i>Tour</i> startet die efa-Tour, eine gründliche Einführung in efa, die jeder Nutzer vor dem Benutzen von efa durcharbeiten sollte! <i>Willkommen</i> zeigt den Willkommens-Bilschirm, <i>efa-Homepage</i> öffnet die efa-Homepage in einem Webbroser. Der Punkt <i><a href="KonsoleFrame.html">Java-Konsole</a></i> zeigt etwaige Fehlerausgaben von efa, und der Punkt <i>Über</i> zeigt ein kurzes Programminfo.
@erklaerung
Bei der Eingabe von Fahrten wurde großer Wert auf Effizienz gelegt. Am schnellsten geht die Eingabe von der Hand, wenn Du mit der &lt;Tab&gt;-Taste von Feld zu Feld springst. efa steuert dabei nur die Felder an, die tatsächlich erforderlich sind. Solltest Du in das vorherige Eingabefeld zurückspringen wollen, so drücke &lt;Shift&gt;-&lt;Tab&gt;. Zum Betätigen eines Buttons, auf dem der Fokus ruht, drücke die Leertaste.
Wenn nacheinander mehrere Fahrten eingegeben werden, bei denen dieselbe Mannschaft im Boot saß oder dieselbe Strecke gerudert wurde (z.B. auf Mehrtagesfahrten), können die Werte der vorherigen Eintrags durch Drücken von &lt;F5&gt; in das aktuelle Eingabefeld übernommen werden.
Wurde für ein Boot eine Standardmannschaft konfiguriert, so kann diese, sobald der Bootsname eingegeben ist, durch Drücken von &lt;F4&gt; in die Felder eingetragen werden.
[but]Erster|Springt zum ersten Eintrag im aktuellen Fahrtenbuch.
[but]&lt;&lt; Vorheriger|Springt zum vorangehenden Eintrag im aktuellen Fahrtenbuch.
[but]Nächster &gt;&gt;|Springt zum nächsten Eintrag im aktuellen Fahrtenbuch.
[but]Letzter|Springt zum letzten Eintrag im aktuellen Fahrtenbuch.
[but]Neu|Beginnt einen neuen Eintrag am Ende des Fahrtenbuchs.
[but]Einf.|Fügt vor dem aktuellen Eintrag einen neuen Eintrag in das Fahrtenbuch ein. Die laufenden Nummern des aktuellen Eintrags und aller folgenden Einträge werden dabei um 1 erhöht.
[but]Suchen|Startet die <a href="SuchFrame.html">Suche</a> nach einem speziellen Eintrag im aktuellen Fahrtenbuch, beginnend mit dem nächsten Eintrag.
[but]Löschen|Löscht den aktuellen Eintrag
[field]gehe zu|Dieses Feld kann benutzt werden, um gezielt zu einem bestimmten Eintrag im Fahrtenbuch zu springen. Eine Eingabe von <i>12</i> beispielsweise veranlaßt efa zu einem Sprung zum Eintrag mit der laufenden Nummer 12 (sofern vorhanden).
[field]Lfd. Nr.|Die laufende Nummer des Eintrags. Sie wird bei neuen Einträgen von efa automatisch vorgeschlagen, kann aber geändert werden. Diese Nummer muß im gesamten Fahrtenbuch eindeutig sein. Wenn mehrere Einträge unter <i>derselben</i> laufenden Nummer abgespeichert werden sollen, kann an die laufende Nummer eine Buchstabe von <i>A</i> bis <i>Z</i> angehängt werden (bei der Ausgabe werden diese Buchstaben ignoriert).
[field]Datum|Das Datum der Fahrt. Bei neuen Einträgen schlägt efa automatisch das Datum der letzten Fahrt vor (z.B. <i>16.10.2002</i>). Soll nun eine Fahrt für den 20. Oktober eingetragen werden, so genügt die Eingabe von <i>20</i>; Monat und Jahr ergänzt efa automatisch. Durch Drücken von &lt;+&gt; bzw. &lt;-&gt; oder &lt;Hoch&gt; bzw. &lt;Runter&gt; kann das Datum einen Tag vor- bzw. zurückgeblättert werden.
[field]Boot|Das Boot. Bei der Eingabe des Anfangs eines Bootsnamens wird dieser mit den bereits in der Bootsliste gespeicherten Namen verglichen. Wird eine Übereinstimmunge gefunden, schlägt efa den ersten passenden Eintrag vor. Durch Drücken von &lt;Runter&gt; bzw. &lt;Hoch&gt; kann zum nächsten bzw. vorherigen übereinstimmenden Eintrag geblättert werden.<br>Wenn der im Feld angezeigte Name in der Bootsliste existiert, erscheint der Button hinter dem Eingabefeld in grün, anderenfalls in rot. Wird das Eingabefeld durch Drücken von &lt;Tab&gt; verlassen, so springt der Cursor im Fall eines roten Buttons nicht in das nächste Eingabefeld, sondern zu dem roten Button. Durch anschließendes Drücken der Leertaste wird der rote Button aktiviert und ein Fenster zum <a href="NeuesBootFrame.html">Hinzufügen des Boots</a> zur Bootsliste öffnet sich. Ein Anklicken eines grünen Buttons zeigt die <a href="NeuesBootFrame.html">Einstellungen zu dem eingetragenen Boot</a> an.
[field]Steuermann|Der Steuermann. Im Gegensatz zu den anderen Ruderern werden die Kilometer des Steuermanns bei der <a href="StatistikFrame.html">Statistikberechnung</a> als Steuerkilometer erfaßt und können getrennt ausgegeben werden.<br>Wenn Du bei den <i>Einstellungen zum Fahrtenbuch</i> unter <i><a href="NeuesFahrtenbuchFrame.html">Administration - Fahrtenbuch</a></i> als Format für Namen <i>Vorname Nachname</i> gewählt hast, mußt Du den Vornamen zuerst angeben, anderenfalls den Nachnamen, gefolgt von einem Komma und dem Vornamen. Die Eingabe eines Steuermanns, sowie das <a href="NeuesMitgliedFrame.html">Hinzufügen</a> dieses, verläuft analog wie für das Feld <i>Boot</i> beschrieben.
[field]Mannschaft 1 ... 24|Die 24 Felder <i>Mannschaft</i> nehmen die Ruderer des aktuellen Eintrags auf. Ihre Kilometer werden im Gegensatz zu denen des Steuermanns bei der <a href="StatistikFrame.html">Statistikberechnung</a> als Ruderkilometer erfaßt und können getrennt ausgegeben werden.<br>Von den 24 Mannschaftsfeldern sind immer nur 8 sichtbar. Zu den anderen acht kann durch Betätigen des Buttons <i>restliche Mannschaft anzeigen</i> umgeschaltet werden. Die Eingabe eines Ruderers, sowie das <a href="NeuesMitgliedFrame.html">Hinzufügen</a> dieses, verläuft analog wie für das Feld <i>Boot</i> beschrieben.
[field]Abfahrt|Die Abfahrtszeit. Sie ist (wie auch die Ankunftszeit) einzig und alleine für die Berechnung der <a href="StatistikFrame.html">Statistikart</a> <i>Dauer</i> und <i>Km/h</i> nötig. Wenn diese Statistiken nicht benötigt werden, kann auf die Eingabe von Abfahrts- und Ankunftszeit verzichtet werden.<br>Bei der Eingabe muß nicht zwangsläufig der Doppelpunkt bemüht werden; eine Eingabe <i>15 30</i> oder <i>1530</i> wird von efa automatisch in <i>15:30</i> umgewandelt.
[field]Ankunft|Die Ankunftszeit. Siehe auch Feld <i>Abfahrt</i>.

@datei=FahrtenbuchNeuFortsetzenFrame.html
@titel=Neues Fahrtenbuch erstellen oder fortsetzen
@beschreibung
Das Fenster <i>Neues Fahrtenbuch erstellen oder Fortsetzen</i> öffnet sich durch Auswahl des Menüpunkts <i>Neues Fahrtenbuch erstellen</i> im Menü <i>Datei</i>. In diesem Fenster gibt es zwei Möglichkeiten, um ein neues Fahrtenbuch zu erstellen.
Ein Fahrtenbuch in efa sollte immer die Fahrten genau eines Jahres aufnehmen. Für ein neues Jahr sollte unbedingt ein neues Fahrtenbuch angelegt werden! Für jedes Fahrtenbuch verwaltet efa eigene Mitglieder-, Boots- und Ziellisten, sowie eine Liste der gespeicherten Statistikeinstellungen. Diese Listen sollten immer nur von einem einzigen Fahrtenbuch benutzt werden, da sich sonst Änderungen an diesen Listen in dem einen Fahrtenbuch auch auf ein anderes Fahrtenbuch auswirken. Beim Fortsetzen eines Fahrtenbuchs (s.u.) werden daher Kopien dieser Listen angelegt, so daß Änderungen an den Listen im neuen Fahrtenbuch die alten Listen des alte Fahrtenbuchs unverändert lassen.
@erklaerung
[but]Neues, unabhängiges Fahrtenbuch mit leere Datenlisten|Über diesen Punkt wird ein gänzlich neues Fahrtenbuch erstellt, das von eventuell bereits existieren Fahrtenbüchern unabhängig ist. Dies bedeutet insbesondere, daß es neue (leere) Boots-, Mitglieder- und Datenlisten erhält. Wähle diesen Punkt, wenn Du zum ersten Mal ein Fahrtenbuch erstellen möchtest. Du gelangst dann zum Fenster <a href="NeuesFahrtenbuchFrame.html">Neues Fahrtenbuch erstellen</a>.
[but]Neues Fahrtenbuch als Fortsetzung des aktuellen Fahrtenbuchs|Über diesen Punkt wird ebenfalls ein neues Fahrtenbuch erstellt. Im Unterschied zum ersten Punkt werden jedoch die Datenlisten des aktuellen Fahrtenbuchs übernommen, d.h. das neue Fahrtenbuch bekommt Kopieren der Mitglieder-, Boots- und Ziellisten des aktuellen Fahrtenbuchs. Wähle diesen Punkt dann, wenn Du ein Fahrtenbuch für ein neues Jahr anlegen möchtest (z.B. 2003), welches das aktuelle Fahrtenbuch (z.B. 2002) fortsetzt.<br>Beim Klicken auf <i>Neues Fahrtenbuch als Fortsetzung des aktuellen Fahrtenbuchs</i> wirst Du anschließend in <i>Schritt 1</i> gefragt, welches Fahrtenbuch fortsetzt werden soll. Im Feld <i>Dateiname</i> ist das aktuelle Fahrtenbuch vorgeschlagen. Wenn Du das aktuelle Fahrtenbuch fortsetzten willst, drücke <i>Weiter</i>, sonst wähle zuerst ein anderes Fahrtenbuch aus.<br>In <i>Schritt 2</i> mußt Du nun den Namen für das neue Fahrtenbuch angeben. @datei=MehrtagestourFrame.html
@titel=Mehrtagesfahrt
@beschreibung
Im Fenster <i>Mehrtagesfahrt</i> werden der Name, sowie einige Eigenschaften von Mehrtagesfahrten festgelegt.
Für die Auswertung des DRV-Fahrtenabzeichens und der DRV Wanderruderstatistik ist es wichtig, daß jede Fahrt, bei der es sich um eine Mehrtagesfahrt handelt, als solche markiert ist. Wähle dazu bei der <a href="EfaFrame.html">Eingabe der Fahrten</a> aus der Liste <i>Art der Fahrt</i> den Namen der Mehrtagesfahrt, wenn diese bereits aufgeführt ist, bzw. den Punkt <i>&gt;&gt;&gt; neue Mehrtagesfahrt</i>. Im zweiten Fall öffnet sich dieses Fenster.
Um eine <a href="AuswahlFrame.html">Liste aller Mehrtagesfahrten</a> des aktuellen Fahrtenbuchs anzuzeigen, wähle aus dem Menü <i>Administration</i> unter <i>Fahrtenbuch</i> den Punkt <i>Mehrtagesfahrten</i>. Durch Auswählen einer Fahrt und Klick auf <i>Bearbeiten</i> öffnet sich dann dieses Fenster.
@erklaerung
Es gibt zwei Möglichkeiten, Mehrtagesfahrten in efa einzutragen. Die eine Möglichkeit besteht darin, jede Etappe der Fahrt einzeln einzutragen. Über das Feld <i>Art der Fahrt</i> bei der <a href="EfaFrame.html">Fahrteneingabe</a> werden diese Etappen dann unter einem gemeinsamen Namen zu einer Mehrtagestour zusammengefaßt.
Die zweite Möglichkeit besteht darin, bei der <a href="EfaFrame.html">Eingabe der Fahrt</a> die gesamte Mehrtagesfahrt als einen einzigen Eintrag hinzuzufügen. Auch in diesem Fall muß die Fahrt über das Feld <i>Art der Fahrt</i> als Mehrtagestour gekennzeichnet werden. Damit efa bei einer vorliegenden Fahrt weiß, ob es sich um die gesamte Fahrt oder nur um eine Etappe handelt, muß die Option <i>Diese Fahrt wird in Form von einzelnen Etappen eingetragen</i> entsprechend gesetzt sein.
Beide Varianten liefern für die <a href="StatistikFrame.html">DRV-Auswertung</a> dasselbe Resultat. Die zweite Variante kann jedoch u.U. Nachteile mit sich bringen: Sie ist <i>ungenauer</i> in der Auswertung mancher Statistiken und erlaubt es auch nicht, während einer Ruderfahrt einzelne Ruderer auszutauschen.
[field]Bezeichnung der Fahrt (Weg und Ziel)|Trage in dieses Feld den Namen der Mehrtagesfahrt ein. Alle Fahrtenbucheinträge, die zu derselben Mehrtagesfahrt gehören, müssen unter diesem einen Namen zusammengefaßt werden. (Aber: Wird eine Tour innerhalb eines Jahres mehrfach gerudert, so handelt es sich um zwei verschiedene Mehrtagesfahrten, für die verschiedene Namen verwendet werden müssen!)
[field]Beginn der Ruderfahrt (Datum)|Der Tag, an dem die Ruderfahrt begonnen hat. Dieses Datum wird u.a. für die Auswertung der Jugend-Fahrten-Abzeichen benötigt. Nach Auffassung des DRV zählt auch der Anreisetag als erster Tag der Ruderfahrt, selbst wenn an diesem Tag nicht gerudert wurde.
[field]Ende der Ruderfahrt (Datum)|Der Tag, an dem die Ruderfahrt aufgehört hat hat. Dieses Datum wird u.a. für die Auswertung der Jugend-Fahrten-Abzeichen benötigt. Nach Auffassung des DRV zählt auch der Abreisetag als letzter Tag der Ruderfahrt, selbst wenn an diesem Tag nicht gerudert wurde.
[field]Anzahl der Rudertage|Trage in dieses Feld bitte die Anzahl der Rudertage ein (ohne Ruhetage). Die hier eingetragene Anzahl an Tagen wird, wenn die Fahrt nicht in Form von Etappen eingetragen wird, <b>jedem</b> Teilnehmer dieser Ruderfahrt gutgeschrieben. Für Jugendfahrten, die von den Teilnehmern für das Erfüllen des DRV-Jugend-Fahrten-Abzeichens benötigt werden, dürfen als Rudertage auch An- und Abreisetage gezählt werden.
[field]Befahrtene Gewässer (durch Kommata getrennt)|Trage in dieses Feld eine durch Kommata getrennte Liste von Gewässern ein, die auf dieser Fahrt erreicht wurden. Diese Angabe wird für das Erstellen der DRV Fahrtenruderstatistik benötigt.
[field]Diese Fahrt wird in Form von einzelnen Etappen eingetragen|Wie bereits in der Einleitung zu diesem Fenster beschrieben, kann eine Ruderfahrt entweder eingetragen werden, indem jede Etappe einzeln eingetragen wird (die Auswahl einer gemeinsamen Mehrtagesfahrt dient dann dazu, diese Etappen zusammenzufassen), oder es kann eine Mehrtagesfahrt als ein einziger Eintrag eingegeben werden. Um diese beiden Möglichkeiten unterscheiden zu können, muß diese Option entsprechend gewählt werden.
[but]OK|Übernimmt die Einstellungen und schließt das Fenster.

@beschreibung
Gruppen dienen dazu, mehrere Mitglieder zusammenfassen zu können.
Basierend auf Gruppen sind Auswertungen möglich, die sich auf alle Mitglieder dieser Gruppe beschränken. Auf diese Weise lassen sich Statistiken für eine Gruppe von Mitgliedern erstellen, die nicht unbedingt den gleichen Status oder ein anderes gemeinsames Kriteirum haben muß.
Des weiteren dienen Gruppen beim Einsatz von efa im Bootshaus dazu, Ruderberechtigungen für einzelne Boote festzulegen. Es lassen sich für jedes Boot bis zu fünf Gruppen konfigurieren. efa prüft dann, ob nur Ruderer dieser Gruppen im Boot sitzen. Außerdem kann eine Gruppe angegeben werden, aus der mindestens ein Ruderer (oder Steuermann) stammt. So läßt sich z.B. festlegen, daß immer Obmann (ein Mitglied der Gruppe Obleute) im Boot sitzt.

[field]Gruppenmitglied|Eine Person, die Mitglied dieser Gruppe ist. Jede Person kann in beliebig vielen Gruppen Mitglied sein.
[but]Mehr Felder|Wenn die Anzahl der Felder zur Eingabe von Gruppenmitgliedern nicht ausreichen, so klicke <i>Mehr Felder</i>, um weitere Eingabefelder hinzuzufügen.

@datei=NeuesZielFrame.html
@titel=Neues Ziel hinzufügen / Ziel bearbeiten
@beschreibung
Dieses Fenster dient zum Festlegen der Entfernung und der Zielbereiche für ein Ziel. Es öffnet sich, wenn Du bei der <a href="EfaFrame.html">Eingabe der Fahrten</a> auf den Button hinter dem Eingabefeld <i>Ziel</i> klickst, oder wenn Du ein Ziel unter <i>Administration - Fahrtenbuch - Zielliste</i> bearbeitest.
Bei Zielen herrscht das Problem, daß oft ein &quot;Grundziel&quot; mit einer Vielzahl verschiedener Abstecher ins Fahrtenbuch eingetragen wird. Es ist sicherlich nicht sinnvoll, jeden Abstecher in die Zielliste aufzunehmen. Häufige Abstecher sollten (schonmal der Einfachheit halber) mit aufgenommen werden, seltene könnten unter dem Zusatz <i>+ Abstecher</i> zusammengefaßt aufgenommen werden oder ganz weggelassen werden.<br>Beispiel: Eigentliche Tour ist die <i>Kleine Umfahrt</i> (16 Km). Ein häufiger Abstecher ist das zusätzliche Umrudern der Pfaueninsel, daher wäre eine Variante <i>Kleine Umfahrt + Pfaueninsel</i> (17 Km) sinnvoll. Seltene Abstecher könnten unter <i>Kleine Umfahrt + Abstecher</i> (17 Km) zusammengefaßt werden. Sollten einzelne dieser Abstecher länger als 17 Km sein, so kann der bei der Fahrt-Eingabe im Feld <i>Boots-Km</i> von efa vorgeschlagene Wert nachträglich korrigiert werden.
<b>Wichtig für Berliner Vereine:</b> Eine Zielfahrt muß <b>immer</b> in die Zielliste aufgenommen werden, d.h. bei jeder Fahrt mit mindestens 20 Km, bei der ein Zielbereich erreicht wird, muß zur Zielliste hinzugefügt werden (auch wenn es sich um einen seltenen Abstecher handelt), denn nur so können efa die erreichten Zielbereiche mitgeteilt werden!<br>Natürlich genügt auch hier die Variante mit <i>+ Abstecher</i>, sofern alle möglichen Abstecher zu einer Fahrt mit mindestens 20 Km und denselben Zielbereichen führen und der Zielbereich für den Abstecher in der Zielliste eingetragen wird. Vorsicht jedoch gilt für obiges Beispiel, für dessen Ziel <i>Kleine Umfahrt + Pfaueninsel</i> (17 Km) <b>kein</b> Zielbereich festgelegt ist. Wird nun eine Kleine Umfahrt mit Abstecher <i>Schwanenwerder Umfahrt</i> gemacht, so wird Zielbereich 2 erreicht und die Fahrt ist länger als 20 Km. Diese Fahrt darf dann nicht unter dem Ziel <i>Kleine Umfahrt + Abstecher</i> eingetragen werden, da für dieses Ziel kein Zielbereich angegeben ist!

Durch Kommata getrennte Liste <b>aller erreichten Zielbereiche</b> (nur für Fahrten mit mindestens 20 Kilometern).<br><b>Hinweis:</b> Es dürfen hier nur <i>erreichte</i> Zielbereiche eingetragen werden! Der Startbereich (der Bereich, in dem losgerudert wurde) darf <i>nicht</i> als Zielbereich eingetragen werden!<br><br>Beispiel: Bei einer Fahrt von Wannsee (Bereich 3) über die Havel-Nord (Bereich 2) nach Tegel (Bereich 1) werden Bereiche <i>1</i> und <i>2</i> erreicht. [reg]Wettbewerbe|zusätzliche Wettbewerbe ausgeben

[field]alle Zielfahrten ausgeben (LRV Berlin Sommer)|Wenn diese Option gewählt ist, gibt efa für Teilnehmer, die den Wettbewerb erfüllen, alle Zielfahrten einer Person aus. @titel=Statistik erstellen
@beschreibung
efa bietet zahlreiche Möglichkeiten, die eingegebenen Fahrten auszuwerten. Es lassen sich verschiedene Arten von Statistiken stellen, sowie die Bedingungen für Kilometerwettbewerbe auswerten. Darüber hinaus kann efa auch Meldedateien für die elektronische Wettbewerbsteilnahme erstellen.
Um in das Fenster <i>Statistik erstellen</i> zu gelangen, wähle aus dem Menü <i>Ausgabe</i> den Punkt <i>Statistiken und Meldedateien erstellen</i>.
@erklaerung
Es gibt drei grundlegend verschiedene Arten von Statistiken: Mannschafts-Kilometer, Boots-Kilometer und Wettbewerbe. Die Einstellungen zu diesen Statistiktypen wird auf den Registerkarten <i><a href="#mannsch">Mannschafts-Kilometer</a></i>, <i><a href="#boot">Boots-Kilometer</a></i> und <i><a href="#wett">Wettbewerbe</a></i> vorgenommen. Die Registerkarte <i><a href="#stat">gespeicherte Statistikeinstellungen</a></i> dient zum Aufrufen von gespeicherten Statistikeinstellungen.
Im Folgenden werden nun zuerst die Felder besprochen, die immer sichtbar sind, und danach die Felder der einzelnen Registerkarten beschrieben.
[field]Zeitraum für die Berechnung|Mittels der Felder <i>vom</i> und <i>bis</i> kannst Du die Auswertung auf einen bestimmten Zeitraum eingrenzen. Wenn diese Felder leer bleiben, werden alle Fahrten ausgewertet.
[field]fahrtenbuchübergreifend|Normlerweise werden nur die Fahrten des aktuellen Fahrtenbuchs ausgewertet. Wenn diese Option gewählt ist, werden auch Einträge aus allen Fahrtenbüchern ausgewertet, die dem aktuellen Fahrtenbuch vorangehen oder das aktuelle Fahrtenbuch fortsetzen (s. <a href="NeuesFahrtenbuchFrame.html">Einstellungen zum Fahrtenbuch</a>). Auf diese Weise lassen sich auch über Jahresgrenzen hinweg Fahrten auswerten.
[field]Ausgabeart|In dieser Auswahlliste kannst Du wählen, in welcher Form die Daten ausgegeben werden. Die verschiedenen Ausgabearten werden im Folgenden kurz erklärt.<br><ul><li><i>im Programm (Grafik)</i>: Die Daten werden graphisch aufbereitet und direkt im Programm angezeigt. Diese Ausgabeart kann auch zum Ausdrucken der Ergebnisse benutzt werden.</li><li><i>im Programm (Text)</i>: Diese Ausgabeart zeigt die ausgewerteten Daten in einer schlichten Tabelle direkt im Programm an.</li><li><i>im Browser</i>: Bei dieser Ausgabeart werden die ausgewerteten Daten in einem beliebigen externen Webbrowser angezeigt (<a href="EfaConfigFrame.html">s. Allgemeine Einstellungen</a>). Auch bei dieser Ausgabeart können die Ergebnisse über die Druckfunktion des Browsers ausgedruckt werden.</li><li><i>als HTML-Datei</i>: Bei dieser Ausgabeart erstellt efa eine HTML-Datei, deren Name im Feld <i>Ausgabe in Datei</i> festgelegt wird. Diese HTML-Datei kann ebenfalls zum Ausdrucken benutzt werden, oder beispielsweise in die Vereins-Homepage eingebunden werden. Um nur die Tabelle mit der Kilometerliste in der Datei zu ersetzen und das Dateigerüst beizubehalten, <a href="StatistikErweitertFrame.html">siehe auch Erweiterte Ausgabeeinstellungen</a>.</li><li><i>als PDF-Datei</i>: Bei dieser Ausgabeart erstellt efa eine PDF-Datei, die sich mit dem <i><a href="http://www.adobe.com/">Adobe</a> Acrobat Reader</i> anzeigen läßt. Auch diese Ausgabeart kann zum Ausdrucken der Ergebnisse benutzt werden. Für die PDF-Ausgabe benötigt efa das XML-Plugin und das PDF-Plugin, welche efa automatisch aus dem Internet laden kann.</li><li><i>als XML-Datei</i>: Diese Ausgabeart erzeugt eine XML-Datei, die beispielsweise für die Weiterverarbeitung der Ausgabedaten durch eine XML-fähige Anwendung benutzt werden kann. Eine DTD für die erzeugte XML-Datei findest Du im <i>ausgabe/layout</i>-Verzeichnis unter dem Namen <i>kilometerliste.dtd</i>. Für die XML-Ausgabe benötigt efa das XML-Plugin, welches efa automatisch aus dem Internet laden kann.</li><li><i>als Textdatei</i>: Bei dieser Ausgabeart erstellt efa eine einfache Textdatei.</li><li><i>als CSV-Datei</i>: Wenn die ausgewerteten Daten mit einer Tabellenkalkulation weiterverarbeitet werden sollen, kann diese Ausgabeart gewählt werden. efa erstellt dann eine Datei mit Feldern, die durch einen Senkrechtstrich getrennt sind (comma separated values). Diese Datei kann von gängigen Tabellenkalkulationen geöffnet werden.</li><li><i>als Meldedatei</i>: Diese Ausgabeart erstellt eine Meldedatei, die zum Melden von Teilnehmern für Kilometerwettbewerbe verwendet werden kann (<a href="VereinsConfigFrame.html">s. auch Elektronische Wettbewerbsmeldung</a>). Diese Ausgabeart steht nur bei Auswahl der Registerkarte <i><a href="#wett">Wettbewerbe</a></i> zur Verfügung.</li></ul>
[field]Formatierung|Aus dieser Liste kann eine Formatierung (Layout) für die Ausgabe ausgewählt werden. Andere Formatierungen als die Standard-Formatierung sind für alle HTML-basierten Ausgabearten möglich (<i>im Programm (Grafik)</i>, <i>im Browser</i>, <i>als HTML-Datei</i>), sowie für die PDF- und die XML-Ausgabe. (Beim Erstellen von Meldedateien kann vom Benutzer kein Namen vorgegeben werden.)<br>Seit Version 1.2.0 kann efa die erstellte Datei auch automatisch auf einen FTP-Server hochladen, um z.B. gleich die Vereins-Homepage zu aktualisieren. In diesem Fall gib in das Feld <i>Ausgabe in Datei</i> den Pfad zur Datei auf dem FTP-Server nach folgendem Schema ein: <tt>ftp://<i>nutzername</i>:<i>passwort</i>@<i>mein.server.de</i>/<i>verzeichnis</i>/<i>auf</i>/<i>dem</i>/<i>server</i>/<i>datei.html</i></tt>. Um diese Eingabe nicht jedes Mal vornehmen zu müssen, kann die Statistik natürlich auch über <i>Statistikeinstellungen speichern</i> gespeichert werden. Hierbei ist zu beachten, daß Nutzername und Paßwort im Klartext (d.h. unverschlüsselt) gespeichert werden! (Für die FTP-Funktionalität benötigt efa ein zusätzliches Plugin, welches beim ersten Benutzen automatisch installiert wird.)

[reg]<a name="mannsch">Mannschafts-Kilometer</a>|Mannschafts-Kilometer
Bei den Mannschafts-Kilometer werden für alle Ruderer/Innen eines Boots die Kilometer berechnet. Grundlage aller Berechnungen sind also die Ruderer, d.h. pro Fahrt werden (außer im Einer) mehrfach Kilometer berechnet --- wenn die Kilometer im Monat ausgewertet werden, ergeben sich also u.U. andere Ergebnisse, als wenn die <a href="#boot">Boots-Kilometer</a> zugrunde gelegt werden.
[field]Statistikart|In diesem Feld wird festgelegt, welche Art von Statistik erstellt werden soll. Es stehen folgende Statistikarten zur Verfügung:<br><ul><li><i>Ruderer/Innen</i>: Es werden die Kilometer für jede(n) Ruderer/in berechnet. Die Leistungen eines jeden Ruderers/in werden einzeln aufgeführt.</li> <li><i>Status</i>: Es werden die Kilometer nach Statusgruppe getrennt berechnet, z.B. Kilometer der Junioren, Senioren usw. Für jede Statusgruppe werden die Leistungen einzeln aufgeführt.</li> <li><i>Jahrgang</i>: Es werden die Kilometer für die einzelnen Jahrgänge berechnet. Die Leistung jedes Jahrgangs wird einzeln aufgeführt.</li> <li><i>Geschlecht</i>: Die Kilometer werden getrennt nach männlichen und weiblichen Personen aufgeführt.</li> <li><i>Ziele</i>: Es werden alle Ziele aufgeführt, zu denen gerudert wurde.</li> <li><i>Km / Fahrt</i>: Es werden die pro Fahrt geruderten Entfernungen ausgegeben.</li> <li><i>Monate</i>: Es wird die Ruderleistung pro Monat ausgegeben.</li> <li><i>Wochentage</i>: Es wird die Ruderleistung pro Wochentag ausgegeben.</li> <li><i>Tageszeit</i>: Es werden die geruderten Kilometer pro Stunde am Tag ausgegeben.</li> <li><i>Boote</i>: Es werden für alle (ausgewählten) Ruderer/innen die Bootskilometer berechnet. Dabei geht beispielsweise ein Vierer mit Steuermann fünfmal in die Bewertung ein (im Unterschied zur Statistik <a href="#boot">Bootskilometer: Boote</a>).</li> <li><i>Bootsart</i>: Gibt die detaillierte Bootsart-Bezeichnung der geruderten Boote aus.</li> <li><i>Fahrtart</i>: Gibt die Art der Fahrt aus.</li> <li><i>Mitruderer/Innen</i>: Es werden die Leistungen aller Personen aufgeführt, die mit einer bestimmten Person, die im Feld <i>nur Name</i> angegeben wird, zusammen gerudert sind.</li> <li><i>Wer mit Wem</i>: Es wird eine Tabelle erstellt, wer mit wem gerudert ist. In den Tabellenfeldern werden alle ausgewählten Daten (z.B. <i>Kilometer</i>, <i>Fahrten</i> und <i>Km / Fahrt</i>) ausgegeben. Zusätzlich kann hier ausgegeben werden, wer mit wie vielen verschiedenen Personen gerudert ist (<i>Anzahl Verschiedene</i>).</li> <li><i>Wer Wohin</i>: Es wird eine Tabelle erstellt, in die für jede(n) (ausgewählte(n)) Ruderer/in ermittelt wird, wohin er/sie gerudert ist. Dabei werden in den Tabellenfeldern alle ausgewählten Daten (z.B. <i>Kilometer</i>, <i>Fahrten</i> und <i>Km / Fahrt</i>) ausgegeben. Zusätzlich kann hier ausgegeben werden, wer zu wie vielen verschiedenen Zielen gerudert ist (<i>Anzahl Verschiedene</i>).</li> <li><i>Wer mit Bootsart</i>: Für jeden Ruderer wird, unterteilt nach der jeweiligen Bootsart, angegeben, wieviel er gerudert ist.</li> <li><i>Fahrtenbuch</i>: Es wird ein persönliches Fahrtenbuch für einzelne Ruderer/Innen bzw. eine Gruppe von Personen ausgegeben.</li> <li><i>Jahre</i>: Diese Statistik faßt alle Fahrten eines Jahres zusammen. So lassen sich mehrere Jahre miteinander vergleichen. Um fahrtenbuchübergreifend die Fahrten aus allen Fahrtenbüchern zu berechnen, muß die Option <i>fahrtenbuchübergreifend</i> aktiviert sein.</li><li><i>Monatsübersicht</i>: Gibt in einer Tagesdarstellung für jeden Monat die geruderten Kilometer aus.</li> <li><i>Wer unerlaubt</i>: Bei dieser Statistikart werden alle Ruderer ausgegeben, die in einem Boot gerudert sind, obwohl sie nicht einer der Gruppen angehörten, die dieses Boot rudern dürfen. Es werden hierbei nur Ruderer (keine Steuerleute) berücksichtigt und auch nur dann ausgegeben, wenn die Anzahl der max. Nicht-Gruppenangehörigen für das jeweilige Boot überschritten ist. Die Gruppe, die für das Boot als <i>mind 1 aus dieser Gruppe</i> konfiguriert ist, findet bei dieser Statistikart keine Berücksichtigung.</li> <li><i>Wer mit Fahrtart</i>: Bei dieser Statistikart wird eine Matrix von Ruderern und ihren Fahrtarten ausgegeben.</li></ul>
[field]Vorjahresvergleich|Ist diese Option gewählt, werden die Ruderleistungen aus dem gewählten Zeitraum mit denen im selben Zeitraum des Vorjahres verglichen und die Veränderungen (plus/minus) ausgegeben. Wird als Zeitraum <i>1.1.2003</i> bis <i>31.12.2003</i> angegeben, so bewirkt dies, daß das Jahr 2003 mit dem Jahr 2002 verglichen wird, und zwar im Zeitraum 1.1. bis 31.12. Werden die Felder <i>vom</i> und <i>bis</i> leer gelassen, so wird das aktuelle Jahr mit dem Vorjahr im Zeitraum vom 1.1. bis zum heutigen Tag verglichen.<br>Hinweis: Da sich die Fahrten des Vorjahres i.d.R. in einem anderen Fahrtenbuch befinden (sollten), muß die Option <i>fahrtenbuchübergreifend</i> ebenfalls gewählt sein!

[field]Art der Fahrt|Es werden nur Fahrten berücksichtigt, deren Art in dieser Liste ausgewählt ist.
[field]Auswahl: Name / Gruppe|Mit dieser Auswahl kann festgelegt werden, ob sich bei dem Eintrag im Feld <i>nur Name/Gruppe</i> um den Namen eines Ruderers oder einer Gruppe handelt.

[field]als Teil eines Namens|Wenn diese Option aktiviert ist, dann wird der unter <i>nur Name</i> angegebene Name nicht als vollständigen Name betrachtet, sondern nur als Namensbestandteil. [field]Ausgabe|Unter <i>Ausgabe</i> kann festgelegt werden, welche Informationen pro Eintrag ausgegeben werden sollen. @datei=SynonymFrame.html
@titel=Synonyme
@beschreibung
efa ermöglicht es, für Mitglieder-, Boots- und Zielnamen sogenannte Synonyme festzulegen. Die Idee ist, daß sich bei der Eingabe von Fahrten mehrere Namen für dasselbe Mitglied/Boot/Ziel verwenden lassen. Bei der Statistik-Auswertung erscheinen dann alle diese Varianten unter einem einheitlichen Namen. 
Bei Booten werden Synonyme zur Realisierung von Kombi-Booten genutzt: In der Bootsliste werden zwei Boote angelegt (z.B. <i>Boot - 4er ohne Stm.</i> und <i>Boot - 3er mit Stm.</i>), die je nach Situation bei der Eingabe benutzt werden. Beide Bootsnamen sind zugleich Synonyme für das Boot <i>Boot</i> (welches sich nicht in der Bootsliste befindet). Bei der Ausgabe werden beide Boote mit dem Namen <i>Boot</i> ausgegeben.
Bei Mitgliedern ist die Verwendung von Synonymen dann sinnvoll, wenn sich der Name eines Mitglieds ändert. Beispielsweise könnte <i>Manuela Musterfrau</i> einen Herrn Mustermann heiraten und anschließend <i>Manuela Mustermann</i> heißen. Damit Manuela auch ihre Kilometer vor ihrer Heirat unter ihrem neuen Namen gutgeschrieben bekommt, kann <i>Manuela Musterfrau</i> als Synonym für <i>Manuela Mustermann</i> festgelegt werden. Ähnlich verhält es sich, wenn ein Gast, der unter dem Namen <i>Kalle Kunze (Gast)</i> geführt wird, plötzlich dem Verein beitritt und nun <i>Kalle Kunze</i> heißt.
Bei Zielen kann die Verwendung von Synonymen sinnvoll sein, wenn verschiedene Ruderer dasselbe Ziel anders nennen. So sind beispielsweise in Berlin <i>Alter Hof</i> und <i>Mückenwiese</i> dasselbe Ziel. Auch hier könnte ein Synonym sinnvoll sein.
Beim Verwenden von Synonymen sollte beachtet werden, daß alle Synonymnamen als Einträge in den jeweiligen Boots-/Mitglieder- bzw. Ziellisten geführt werden sollten. Kalle sollte also sowohl als <i>Kalle Kunze (Gast)</i>, als auch als <i>Kalle Kunze</i> in der Mitgliederliste eingetragen sein. Auf diese Weise werden beispielsweise seine Kilometer unter dem Namen <i>Kalle Kunze (Gast)</i> als Gast-Kilometer gewertet. Wenn die Berechnung der Einträge für Gäste ausgeschlossen wird, werden nur seine Kilometer als Mitglied berechnet. Der Hauptname zu einem Synonymnamen kommt also erst bei der Ausgabe des Namens zum Tragen. Für die Entscheidung, welche Einträge berechnet werden sollen usw. benutzt efa die Daten zu dem Namen, so wie er konkret vorliegt.
Um ein Synonym hinzuzufügen oder zu verändern, wähle aus dem Menü <i>Administration</i> den Punkt <i>Synonymlisten</i>, und anschließend <i>Mitglieder</i>, <i>Boote</i> oder <i>Ziele</i>.

Wenn eine schreibgeschützte Datei geschrieben werden soll, wirst Du gefragt, ob Du den Schreibschutz übergehen oder aufheben möchtest. Im ersten Fall wird der Schreibschutz nur für dieses eine Mal übergangen, im zweiten Fall wird der Schreibschutz permanent entfernt. Du wirst dann nicht mehr nach dem Paßwort befragst, bis Du efa das nächste Mal startest. (Das hier eingegebene Paßwort merkt sich efa selbstverständlich nur, bis efa beendet wird!)

Jede der Dateien läßt sich durch ein Häkchen für die Aktionen <i>Schreibschutz aktivieren</i> und <i>Schreibschutz deaktivieren</i> markieren (standardmäßig sind alle Dateien bis auf die efa-Konfigurationsdatei ausgewählt).

@beschreibung
eddi, der efa Datenlisten-Importeur, ist ein Zusatzprogramm von efa, welches dazu dient, Datenlisten (Mitglieder-, Boots- und Ziellisten) aus einer Tabellenkalkulation in efa zu importieren.
Um Datenlisten nach efa zu übernehmen, gehe folgendermaßen vor:<br>Wenn noch kein Fahrtenbuch in efa erstellt wurde, so erstelle nun zunächst ein neues Fahrtenbuch (<a href="NeuesFahrtenbuchFrame.html">Neues Fahrtenbuch erstellen</a>). Wenn Du das Fahrtenbuch <i>2002.efb</i> genannt hast, so nennt efa die dazugehörigen Listen standardmäßig <i>2002.efbm</i> (Mitgliederliste), <i>2002.efbb</i> (Bootsliste) und <i>2002.efbz</i> (Zielliste). Nach dem Erstellen des Fahrtenbuchs und der dazugehörigen Datenlisten schließe efa nun wieder.
Als nächstes müssen die zu importierenden Datenlisten von Deinem Tabellenkalkulationsprogramm als <i>CSV-Datei</i> (<i>comma separated values</i>) exportiert werden. Öffne dazu die Datei in der Tabellenkalkulation, wähle aus dem Menü <i>Datei</i> den Punkt <i>Speichern unter...</i> und wähle als <i>Dateityp</i> den Punkt <i>CSV-Datei (*.csv)</i> aus. Speichere die Liste nun ab (z.B. unter dem Namen <i>mitglieder.csv</i>). (Je nach verwendeter Software können die Bezeichnungen etwas variieren, u.U. kommt auch noch ein Hinweis, daß bei diesem Format Inhalt verloren gehen könnte.) Diese CSV-Datei dient als Eingabe für eddi.
Starte nun eddi über den Eintrag im Startmenü und wähle die im vorigen Schritt erstellte CSV-Datei über das <i>Datei öffnen</i>-Symbol aus. Falls die blau unterlegt angezeigte Zeile nur eine Überschrift o.ä. enthält, springe mit dem Button <i>nächste</i> bis zu der ersten Zeile, die einen richtigen Eintrag enthält. In der Regel erkennt eddi das verwendete Trennzeichen, welches die einzelnen Felder voneinander trennt, automatisch. Sollte eddi ein falsches Zeichen erkannt haben, so gib im Feld <i>Trennzeichen</i> das korrekte Trennzeichen (z.B. einen Senkrechtstrich |) ein.<br>Wähle nun im rechten Bereich des Fensters aus, ob es sich bei der zu importierenden Liste um eine Mitglieder-, Boots- oder Zielliste handelt und ordne anschließend den einzelnen Werten (<i>Vorname</i>, <i>Nachname</i> usw.) jeweils ein Feld aus der CSV-Datei zu. Sollten manche Werte in der CSV-Datei fehlen, so ist keine Zuordnung erforderlich. Nachdem alle Werte zugeordnet wurden, klicke auf <i>Datenliste importieren</i>, um den Import zu beginnen. Als Dateinamen gib den Namen der (leeren) von efa im ersten Schritt erstellten Liste an (z.B. <i>2002.efbm</i>, falls es sich um eine Mitgliederliste für das Fahrtenbuch <i>2002.efb</i> handelt). Beim Import von unvollständigen oder fehlerhaften Einträgen fordert eddi automatisch zum Vervollständigen auf.
@erklaerung

[but]einlesen|Die Datei, deren Name im Feld <i>Dateiname</i> angegeben ist, neu einlesen.
[but]vorherige|Eine Zeile zurückspringen. Wähle mit den Buttons <i>vorherige</i> und <i>nächste</i> die erste Zeile aus, die die zu importierenden Datensätze enthält. Der Import wird mit der ausgewählten Zeile begonnen.
[but]nächste|Eine Zeile vorspringen. Wähle mit den Buttons <i>vorherige</i> und <i>nächste</i> die erste Zeile aus, die die zu importierenden Datensätze enthält. Der Import wird mit der ausgewählten Zeile begonnen.
[field]Trennzeichen|Gib in dieses Feld das Zeichen ein, welches die einzelnen Felder in der zu importierenden Datei trennt. Normalerweise erkennt eddi das Trennzeichen automatisch.
[but]Datenliste importieren|Startet den Import der Datenliste, wenn alle Werte den entsprechenden Feldern zugeordnet wurden. Als erstes wird nach dem Dateinamen der zu erstellenden Liste gefragt: Gib hier bspw. <i>2002.efbm</i> für eine Mitgliederliste für das Fahrtenbuch <i>2002.efb</i> an (siehe einleitende Erklärung).<br>Bei Mitgliederlisten wird zusätzlich nach der Liste der Status gefragt, die ein Mitglied im Verein haben kann. Diese muß mit der Liste, wie sie beim Erstellen des Fahrtenbuchs in efa angegeben wurde, übereinstimmen.

@beschreibung
eddi, der efa Datenlisten-Importeur, ist ein Zusatzprogramm von efa, welches dazu dient, Datenlisten (Mitglieder-, Boots- und Ziellisten) aus einer Tabellenkalkulation in efa zu importieren.
@erklaerung
[but]Schließen|Schließt dieses Fenster.

EddiFrame.html|eddi - efa Datenlisten-Importeur

@datei=MitgliederFehlerFrame.html
@titel=Fehler in zu importierendem Eintrag (Mitglieder)
@beschreibung
Dieses Fenster erscheint, wenn beim Import von Mitgliederlisten mit <a href="EddiFrame.html">eddi</a> ein Eintrag gefunden wurde, der unvollständig oder inkorrekt ist. In diesem Fenster kann der Eintrag korrigiert werden.
@erklaerung
[field]Vorname|Vorname der Person.
[field]Nachname|Nachname der Person.
[field]Eingabekürzel|Kürzel, welches zur schnelleren Eingabe (anstelle des Namens) verwendet werden kann.
[field]Jahrgang|Geburtsjahr der Person.
[field]Geschlecht|Geschlecht der Person.
[field]Status|Status der Person im Verein.
[field]Verein|Verein der Person, falls es sich um eine Person eines fremden Vereins handelt. Bei Mitgliedern sollte dieses Feld leer bleiben.
[field]Behinderung|Legt fest, ob die Person einen Behinderungsgrad von 50 oder mehr Prozent hat (für DRV-Fahrtenabzeichen nötig).
[field]'abc' immer durch 'xyz' ersetzen|Wenn diese Option aktiviert wird, so wird auch bei allen weiteren Einträgen in diesem Feld der Text <i>abc</i> durch <i>xyz</i> ersetzt. Dies ist praktisch, wenn bspw. in der zu importierenden Datei das Geschlecht mit <i>m</i> bzw. <i>w</i> angegeben ist, efa aber <i>männlich</i> bzw. <i>weiblich</i> verlangt. Wenn eddi einmal weiß, daß <i>m</i> durch <i>männlich</i> zu ersetzen ist, fragt er beim nächsten Mal nicht erneut nach.
[but]OK|Änderungen an dem Eintrag übernehmen und mit Import fortfahren.
@siehe
EddiFrame.html|eddi - efa Datenlisten-Importeur
@ende
